The Pixelated Miner

Built with GameMaker Studio

Download Project

Project Summary

The Pixelated Miner is a 2D video game programmed in GML, drawing inspiration from popular titles such as Terraria and Minecraft. Players must excavate and collect randomly generated ores scattered throughout the game's world map, with the objective of amassing 100 points within a 60-second time limit. The game's intuitive controls provide a user-friendly experience, while the use of procedurally generated ore deposits ensures a unique and dynamic gameplay experience with each playthrough

Features and functionalities

    Torches

  • When a player places a torch, it emits light in a circular area around it, illuminating the surrounding area and making it easier to navigate.

    • ladders

  • Ladders are a versatile tool in The Pixelated Miner that allow players to climb up and down walls, making it easier to explore the underground caves.

    • Lighting system

  • As players move through the underground caves, the lighting shifts to reflect their position and create a sense of depth and atmosphere.

    • Random Resource Generation

  • The Pixelated Miner uses a random resource generation system to create a dynamic and changing underground world. In this system, resources are generated randomly throughout the game world, with the most common ores appearing near the surface and rarer resources appearing deeper underground.

Technologies used

Aseprite

This pixel art editor was used to create the game's sprites and animations. Its user-friendly interface and powerful drawing tools enabled the creation of pixel art assets that helped to complete the game.

GameMaker Studio 1.4

This game engine was used to develop the core gameplay mechanics, including character movement and resource collection. The engine's built-in code editor allowed for rapid prototyping and iteration during the development process.

GameMaker Studio Debugger

The Gamemaker Studio 1.4 Debugger was an essential tool in the development of The Pixelated Miner. It allowed me to step through the game's code, line by line, and quickly identify and fix any errors or bugs in the code.

Bfxr

The tool allows users to create unique sound effects by adjusting various parameters such as frequency, waveform, and volume. In the project, Bfxr was used to create the sound effect for the button when clicked.

Challenges and solutions

Random Resource and Terrain Generation

Generating the resources

How do I go about creating an algorithm to generate random resources spread out in the game world, controlling the quantity and depth of each type of resource? Well, one solution I found was to use the irandom_range function to generate a random number between, for example, 200-300. Using the irandom_range function, I could generate random numbers for the x and y positions to be used to decide where the game will place an instance of the resource. Along with using the instance_number function to control the amount that will be created.

The lighting system

One of the challenges I faced while developing a lighting system was dynamically adjusting the lighting based on the player's actions, such as placing torches or moving. To overcome this, I utilized GameMaker's surface tool, which acts like a blank canvas. By drawing a black rectangle and using blend modes, I was able to cut out a circle from the rectangle, which allowed me to adjust the lighting based on the player's actions.
